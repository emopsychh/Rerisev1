# ReRise — PRD v0.2

> **Синхронизация 15.07.2026:** актуальный бренд — ReRise. Утверждённые тарифы и финансовые правила находятся в [`MARKETING_PLAN.md`](./MARKETING_PLAN.md); продуктовые права тарифов, цены/PV курсов и токенов пока не утверждены.

## Контекст

ReRise — это AI-платформа с обучением, личным кабинетом, магазином, подписками, токенами и партнерской моделью.

Главная продуктовая формула:

```txt
ReRise = AI Workspace + Academy + Store + Partner System + Materials + Admin
```

ReRise не должен быть копией MetaBox. Конкурентская структура полезна как референс: программы, AI Hub, партнерская механика, материалы, чат партнеров, биржа труда. Но ядро ReRise должно быть сильнее: AI-инструмент не отдельно, обучение не отдельно, партнерка не отдельно. Все связано в одну рабочую систему.

## Видение

Пользователь ReRise может:

- работать с нейросетями в едином AI-интерфейсе;
- проходить обучение по GPT, визуалу, видео, контенту, автоматизациям и цифровым продуктам;
- покупать программы, подписки и токены;
- получать доступы, ключи, бонусы и материалы;
- приглашать других пользователей через партнерскую систему;
- видеть команду, начисления, историю покупок и прогресс.

Ключевая идея: не продавать "курс по нейросетям", а дать человеку рабочую AI-среду, где он учится, применяет, сохраняет результат, монетизирует навык и может продвигать платформу как партнер.

## Цели

### Бизнес-цели

- продажа цифровых продуктов: курсы, пакеты, промпты, подписки;
- регулярная выручка через AI Workspace и токены;
- рост через партнерскую модель;
- удержание через историю, проекты, обучение и токены;
- масштабируемость для новых программ и AI-моделей.

### Пользовательские цели

- новичок: понять, как использовать AI без хаоса;
- блогер/эксперт: быстрее делать контент, визуал и продукты;
- предприниматель: автоматизировать задачи бизнеса;
- маркетолог/дизайнер: ускорить работу с текстами, визуалом и креативами;
- партнер: продавать продукт и видеть прозрачные начисления;
- куратор: проверять задания и помогать ученикам;
- админ: управлять пользователями, оплатами, курсами и бонусами.

## Позиционирование

Короткая версия:

**ReRise — платформа, где нейросети, обучение и партнерская система собраны в одном личном кабинете.**

Расширенная версия:

**ReRise помогает людям осваивать AI через практические программы и сразу применять навыки в рабочем AI-пространстве: генерировать тексты, изображения, видео, промпты, продукты и материалы для бизнеса. Платформа также дает партнерам инструменты для продвижения, командного роста и прозрачного учета бонусов.**

## Отличие от MetaBox

| MetaBox | ReRise |
| --- | --- |
| Карточки программ + AI Hub | Единый AI Workspace как ядро |
| Обучение отдельно | Обучение встроено в рабочие сценарии |
| Партнерка заметно доминирует | Продуктовая ценность стоит первой |
| Пакет программ | Пакет навыков + инструмент + система применения |
| Калькулятор бонусов | Bonus ledger, история, расчетные периоды |
| Материалы для партнеров | Материалы + готовые AI-воронки + шаблоны |

Утверждённые партнёрские продукты:

- Rise — $90, максимум 30 PV, глубина 3 уровня;
- Rise Pro — $300, максимум 90 PV, глубина 9 уровней;
- Rise Pro Max — $900, максимум 300 PV, глубина 15 уровней;
- ежемесячное продление со второго месяца — $30 и 9 PV в пределах тарифной глубины получателя.

Каталог программ, token packs, их цены, PV и включение в тарифы пока не утверждены. Эти данные нельзя переносить из демонстрационного интерфейса в production-каталог.

Покупка:

```txt
product selected
→ checkout
→ payment created
→ payment paid
→ order completed
→ access granted
→ PV created
→ referral bonus calculation queued
→ notification sent
```

## Tokens

Токены нужны для текстовых запросов, генерации изображений, обработки файлов, видео и дополнительных моделей.

Списания должны идти через ledger:

```txt
TokenTransaction → recalculated balance
```

Нельзя делать:

```txt
user.tokens = user.tokens - 5
```

## Partner Cabinet

Партнерский кабинет должен показывать:

- личный код;
- реферальную ссылку;
- наставника;
- личные продажи;
- количество приглашенных;
- PV;
- бонусы за период;
- доступный баланс;
- pending-баланс;
- кнопку запроса выплаты.

Утверждённые типы начислений:

```txt
direct_purchase
direct_subscription
binary
matching
quick_start
rank
adjustment / reversal
```

Backend можно вводить поэтапно, но интерфейс не должен имитировать финальное начисление без ledger-события и утверждённой версии правила.

## Bonus Engine

Бонусы нельзя считать из текущего состояния интерфейса. Нужен отдельный ledger-based модуль:

```txt
purchase
→ order paid
→ PV ledger entry
→ bonus period
→ bonus calculation
→ bonus accrual
→ payout request
→ payout completed
```

Критичные сущности:

- PVLedgerEntry;
- BonusRule;
- BonusPeriod;
- BonusRun;
- BonusAccrual;
- WithdrawalRequest.

## Нефункциональные требования

- первая загрузка dashboard до 2 секунд;
- AI chat должен показывать streaming-ответ;
- тяжелые задачи через queue;
- изображения и файлы через object storage;
- RBAC;
- audit log;
- rate limiting;
- защита payment webhooks;
- все платежи через Order/Payment;
- все токены через TokenLedger;
- все бонусы через BonusLedger;
- bonus rules версионируются.

## Что не входит в MVP

- сложная биржа труда;
- маркетплейс специалистов;
- нативные мобильные приложения;
- десятки AI-моделей;
- неутверждённые параметры автоматического вывода (сеть, комиссия, SLA, лимиты, KYC/2FA);
- любые дополнительные ранги и бонусы вне утверждённой лестницы из 16 статусов;
- глубокая геймификация;
- соцсеть внутри платформы.
